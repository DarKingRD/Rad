# Frontend часть RadPlan

Это клиентская часть приложения RadPlan, разработанная с использованием современных веб-технологий.

## Технологический стек

- **React 18** - библиотека для создания пользовательских интерфейсов
- **TypeScript** - строгая типизация для повышения надежности кода
- **Vite** - быстрая система сборки
- **Tailwind CSS** - утилитарный CSS-фреймворк для быстрой разработки интерфейсов
- **Recharts** - библиотека для создания интерактивных графиков
- **Lucide React** - набор иконок
- **Axios** - HTTP-клиент для взаимодействия с API

## Структура проекта

```
frontend/
├── public/              # Статические файлы
├── src/                 # Исходный код приложения
│   ├── components/      # Компоненты пользовательского интерфейса
│   ├── data/            # Моковые данные для разработки
│   ├── services/        # Сервисы для работы с API
│   ├── types/           # TypeScript типы
│   ├── App.tsx         # Корневой компонент приложения
│   └── main.tsx         # Точка входа в приложение
├── index.html           # HTML-шаблон
├── package.json        # Зависимости и скрипты
├── tsconfig.json       # Конфигурация TypeScript
├── vite.config.ts      # Конфигурация Vite
└── tailwind.config.js  # Конфигурация Tailwind CSS
```

## Компоненты приложения

### Основные разделы

1. **Дашборд** - главная страница с обзорной статистикой
2. **Планирование смен** - управление расписанием врачей
3. **Распределение исследований** - назначение исследований врачам
4. **Врачи** - справочник врачей с информацией о загрузке
5. **Отчеты** - аналитика по выполнению исследований

### Компоненты интерфейса

- **Header** - верхняя панель с датой и кнопкой обновления
- **Sidebar** - боковое меню навигации
- **KPICard** - карточки с ключевыми показателями
- **DashboardView** - основное представление дашборда
- **ShiftPlanningView** - планирование смен
- **CurrentDistributionView** - распределение исследований
- **DoctorsView** - список врачей
- **ReportsView** - отчеты и аналитика

## Работа с API

Все данные в приложении получаются через REST API бэкенда. Взаимодействие с API реализовано через сервис `api.ts`, который использует Axios для выполнения HTTP-запросов.

Основные эндпоинты:
- `/api/doctors/` - врачи
- `/api/studies/` - исследования
- `/api/schedule/` - расписание
- `/api/study-types/` - типы исследований
- `/api/dashboard/stats/` - статистика для дашборда
- `/api/chart/data/` - данные для графиков

## Стилизация

Для стилизации используется Tailwind CSS - утилитарный CSS-фреймворк, который позволяет быстро создавать современные интерфейсы. Все стили задаются через классы в JSX-компонентах.

## Сборка и развертывание

Для сборки проекта используется Vite, который обеспечивает быструю сборку и горячую перезагрузку во время разработки.

Основные команды:
```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки
npm run dev

# Сборка для продакшена
npm run build

# Предварительный просмотр продакшен-сборки
npm run preview
```

## Типизация

В проекте используется TypeScript для обеспечения типобезопасности. Все типы данных определены в файле `types/index.ts`, что облегчает поддержку и развитие приложения.
